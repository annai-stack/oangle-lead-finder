Lead Finder

One command turns a target segment into a research-backed, contact-level sales spreadsheet — ranked by how hot the lead is.

Built by AKIN for Oangle restaurant technology
The problem

Good outbound dies in the research.

Finding the right F&B chains, the real decision-makers inside them, and a credible reason to reach out is hours of manual work per lead — so reps either send generic blasts or never send at all.

  • Who — which multi-outlet operators actually fit the product?
  • Whom — the owner / COO / IT lead, with a real email and LinkedIn?
  • Why now — a personalised angle, not a templated cold open?
The solution

A pipeline that researches like an SDR — then checks its own work.

Point it at a segment (or an existing account list). It sources companies, finds decision-makers, deep-researches each one with Claude + live web search, and exports the finished 23-column outreach sheet.

  • Contact-level, not company-level — real names, emails, LinkedIn.
  • Personalised — a tailored sales paragraph and value-prop fit per person.
  • Prioritised — every lead carries a 1A→4 heat rank from a fixed rubric.

Inputs · what you feed it

Four inputs. Three are set once.

1 · API keys once

  • Anthropic — enrichment & polish (required)
  • Hunter — real emails / LinkedIn (required)
  • Apollo — firmographics, company size (required)
  • BuiltWith — tech signal (optional)

2 · Company source per run

  • Existing CSV — reuse a prior account list (free, no search)
  • Fresh web search — discover new companies by segment

3 · Run configuration per run

  • Max companies · contacts per company
  • Title & seniority filters · contact source
  • Draft / premium model · batch mode

4 · Client profile once

  • ICP, target titles, value prop, scoring rubric
  • New client = one dict entry, no code changes

Trust · verification layer

"Research-backed" means checked — not just claimed.

Email verified

  • Every address scored deliverable / risky / undeliverable
  • Undeliverable is never auto-sent — protects sender reputation

Source cross-checked

  • The discovery source URL must actually resolve
  • Claude re-checks the page supports the company & outlet claims

Confidence objective

  • High / Med / Low from real signals — not the model's self-report
  • Low-confidence rows routed to a Review tab
Closes the gap that kills outbound trust — a sheet of confident-but-wrong facts. Reps see what's solid and what to check first.
